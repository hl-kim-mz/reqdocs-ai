## 1. 서비스 정의

**ReqDocs AI**는 기획팀의 개발 프로세스 자동화를 위한 웹 서비스입니다.  
회의록·이메일·메모 등 비정형 입력을 그대로 넣으면, AI가 요구사항을 구조화하고  
기획자 검토를 거쳐 개발에 필요한 정형 문서 세트를 자동 생성합니다.

### 핵심 가치

```
[비정형 입력]
 회의록 / 이메일 / 기획 메모 / 인터뷰 노트 / 파일 업로드
          ↓
[AI 요구사항 구조화]          ← 핵심 단계
 기능/비기능/제약사항 분류
 MoSCoW 우선순위 자동 배정
          ↓
[기획자 검토 · 보완]          ← Human Checkpoint
 항목 추가·삭제·우선순위 조정
          ↓
[개발 문서 자동 생성]
 PRD · 기능 명세서 · API 문서 · ERD
          ↓
[내보내기]
 Markdown / PDF / DOCX
```

> **기존 도구와의 차이**: 기존 문서 생성 도구는 "이미 정리된 요구사항"을 전제합니다.  
> ReqDocs AI는 **정리되지 않은 입력에서 시작**하여 구조화→검토→생성의 전 흐름을 하나의 워크스페이스에서 처리합니다.

## 2. 메인 플로우 (3단계)

### Step 1 — 비정형 입력

| 입력 방식 | 설명 |
|-----------|------|
| 텍스트 직접 입력 | 회의록, 이메일, 메모 붙여넣기 (최대 10,000자) |
| 파일 업로드 | `.txt` `.md` `.pdf` `.docx` (최대 10MB) |
| (Post-MVP) URL | Notion, Google Docs 링크 직접 연결 |

### Step 2 — AI 요구사항 구조화 + 기획자 검토

AI가 입력을 분석하여 아래 구조로 자동 변환합니다.

```json
{
  "id": "REQ-001",
  "category": "기능 요구사항",
  "title": "사용자 로그인",
  "description": "이메일/비밀번호로 로그인, 소셜 로그인(Google) 지원",
  "priority": "Must Have",
  "source": "회의록 3페이지 — '인증은 필수'"
}
```

**기획자 검토 화면**에서:
- 카테고리별 요구사항 목록 확인
- MoSCoW 우선순위 태그 조정
- 항목 추가·삭제·설명 인라인 편집
- [재분석] 또는 [문서 생성 →] 선택

### Step 3 — 개발 문서 자동 생성

구조화된 요구사항을 기반으로 선택한 문서 종류를 병렬 생성합니다.

| 문서 | 설명 | 형식 |
|------|------|------|
| PRD | 제품 요구사항 정의서 | Markdown |
| 기능 목록 | Feature List + MoSCoW 우선순위 | 테이블 |
| 기능 명세서 | 기능별 상세 정의 (조건·예외 포함) | Markdown |
| API 문서 | Endpoint 목록 + Request/Response | OpenAPI 3.0 |
| ERD | 엔티티 관계 다이어그램 | Mermaid |

## 3. 사용자 시나리오

### 주요 페르소나

| 페르소나 | 입력 방식 | 기대 산출물 |
|---------|-----------|------------|
| **서비스 기획자** | 클라이언트 미팅 회의록 붙여넣기 | 구조화된 요구사항 목록 + PRD 초안 |
| **개발 PM** | 기능 요청 이메일 스레드 업로드 | API 문서 + ERD 초안 |
| **스타트업 창업자** | 아이디어 메모 자유 입력 | 전체 문서 세트 (개발팀 전달용) |

### Happy Path

```
1. 비정형 텍스트 붙여넣기 or 파일 업로드
2. [요구사항 구조화 시작] 클릭 → AI 분석 (스트리밍)
3. 구조화 결과 검토 화면에서 항목 확인·수정
4. 생성할 문서 종류 선택 → [문서 생성] 클릭
5. 생성 진행 확인 (실시간 스트리밍)
6. 탭 전환으로 각 문서 확인 및 편집
7. Markdown / PDF / DOCX 내보내기
```

## 5. 성공 지표

| 지표 | 목표 |
|------|------|
| 요구사항 구조화 정확도 | 기획자 수정률 < 30% (항목의 70% 이상을 그대로 사용) |
| 구조화 소요 시간 | 일반 회의록 기준 < 15초 |
| 문서 생성 소요 시간 | 전체 5종 기준 < 60초 |
| PRD 도메인 일치율 | > 80% |
| API 응답 시간 | 문서 조회 < 200ms |
